```python
# lines 24-56 — http() with 429/503 Retry-After backoff, and arcgis() wrapper
def http(url, params=None, tries=3):
    if params:
        url = url + "?" + urllib.parse.urlencode(params)
    delay = 2
    for i in range(tries):
        try:
            req = urllib.request.Request(url, headers=UA)
            return urllib.request.urlopen(req, timeout=60).read()
        except urllib.error.HTTPError as e:
            if e.code in (429, 503):
                ra = e.headers.get("Retry-After")
                wait = int(ra) if ra and ra.isdigit() else delay
                if i + 1 == tries: raise
                time.sleep(wait); delay *= 4
            elif 400 <= e.code < 500:
                raise
            else:
                if i + 1 == tries: raise
                time.sleep(delay); delay *= 4
        except Exception:
            if i + 1 == tries: raise
            time.sleep(delay); delay *= 4
UA = {"User-Agent": "Mozilla/5.0 (RadarFundiario; uso pessoal; contato do dono do repo)"}
# arcgis(params) forces f=json, raises RuntimeError if response has an "error" key
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| third-party ArcGIS server → build script | Untrusted JSON from a public endpoint the project does not control, parsed at build time |
| generated artifact → repo | Static file committed into the repo, later loaded by the app (Phase 3) |

## STRIDE Threat Register

This phase's attack surface is an OFFLINE, one-shot build script doing READ-ONLY HTTPS GET requests to a PUBLIC, unauthenticated ArcGIS endpoint and writing a static file into the repo. The generated artifact contains ONLY public bairro geometry + names (`id`, `nm_bai`) — no personal data, and crucially never `dtnascimen` or any cadastral PII (layer 2 has none). There are NO high-severity threats in this phase. Stated explicitly rather than invented.

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | JSON parsed from third-party ArcGIS (compromised/malicious upstream could inject a payload in `nm_bai`) | mitigate | Parse defensively with `json.loads` only (no `eval`/JSONP/dynamic code — matches atualizar-caixa.py); validate shape before trusting (FeatureCollection of Polygons, count assertion). Rendering of `nm_bai` is out of scope this phase; Phase 3 MUST reuse the existing `esc()`/`textContent` escaping — flagged, not fixed here. |
| T-01-02 | Denial of Service | Fragile ArcGIS endpoint (502 under load) | accept | One-time offline build with polite paging (page_size=500) and existing 429/503 Retry-After backoff from atualizar-caixa.py. Not a runtime dependency; a transient failure just means re-run the script. |
| T-01-03 | Information Disclosure | Secrets / PII leaking into the committed artifact | mitigate | Artifact contains only public geometry + `id`/`nm_bai`. No secrets are read or written (public endpoint, no auth, no env vars). `outFields` is restricted to `id,nm_bai` — `dtnascimen` and other PII fields are never requested. |
| T-01-04 | Integrity (silent truncation) | Trusting the unpaginated default response as "complete" | mitigate | Explicit `resultOffset` pagination + hard `assert len(features)==returnCountOnly` — the script aborts non-zero rather than commit a truncated ("furado") artifact, matching atualizar-caixa.py's abort-loudly precedent. |
</threat_model>
